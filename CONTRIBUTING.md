# Contributing

Thank you for considering contributing to CustomInventoryLib.

## How to contribute

1. Fork the repository
2. Create a branch: `git checkout -b fix/your-fix-name`
3. Make your changes
4. Make sure the code follows the existing style (strict types, no comments in code)
5. Submit a pull request with a clear description of what you changed and why

## Code style

- `declare(strict_types=1)` in every file
- No inline comments in code
- PHP 8.1+ features are welcome (enums, readonly, etc.)
- Keep public API backwards compatible

## Reporting bugs

Use the [bug report template](.github/ISSUE_TEMPLATE/bug_report.md).
Always include the client version(s) where the bug occurs, as behavior differs between legacy (1.21.x) and modern (26.x+) Bedrock clients.

## Questions

Open an issue or reach out directly.
